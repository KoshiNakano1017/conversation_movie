# 要求仕様書
## ConversationMovie - Google Meet会話動画自動生成サービス

**文書バージョン**: 1.0.0  
**作成日**: 2026-05-19  
**ステータス**: Draft

**更新メモ（2026-05-23）**: 入力経路は音声ファイルだけでなく、トランスクリプトファイル（`.txt` / `.md` / `.srt` / `.vtt`）のアップロードにも対応する。

---

## 1. プロジェクト概要

### 1.1 サービス名
**ConversationMovie**

### 1.2 サービスコンセプト
Google Meetでの会話を録音・AIで分析し、かわいいアバターが語る解説動画を自動生成してYouTubeに投稿するサービス。会議の価値を最大化し、知識のコンテンツ化を自動化する。

### 1.3 ターゲットユーザー
| セグメント | 説明 |
|---|---|
| 一次ユーザー | Google Meetを日常的に使う個人・フリーランス・スタートアップ |
| 二次ユーザー | 会議内容をコンテンツとして発信したいYouTuber・ポッドキャスター |
| 三次ユーザー | 社内知識共有を自動化したい小規模チーム |

### 1.4 ビジネス目標
- **個人開発MVP**: 3ヶ月以内にコア機能を稼働
- **月間コスト**: 無料枠活用で$50以下に抑制
- **価値提供**: 会議1時間 → 動画コンテンツ化まで15分以内に自動化

## 2. 機能要求

### 2.1 必須機能（Must Have）

#### F-001: Google Meet録音
- Google Meetの音声をリアルタイムまたは事後に取得する
- 対応形式: WAV / MP3 / MP4（映像付き）
- 最大収録時間: 3時間

#### F-002: 音声アップロード
- ウェブUIから音声ファイルまたはトランスクリプトファイルをアップロード
- ドラッグ&ドロップ対応
- 対応形式:
  - 音声: mp3, wav, mp4, m4a, ogg, webm
  - トランスクリプト: txt, md, srt, vtt
- 最大ファイルサイズ: 500MB

#### F-003: Whisperによる文字起こし
- OpenAI Whisper（ローカル実行）で音声→テキスト変換
- 対応言語: 日本語・英語（自動検出）
- タイムスタンプ付き文字起こし
- 精度目標: 日本語WER 15%以下

#### F-004: 話者分離（Speaker Diarization）
- 複数話者の発言を話者ラベルで分離（Speaker A, Speaker B...）
- pyannote.audioを使用
- 話者数: 最大10名

#### F-005: Gemini APIによる要約
- 文字起こしテキストをGemini APIで要約
- 要約レベル: 短文(3行)・中文(段落)・詳細(構造化)
- 会話テーマの自動抽出

#### F-006: 会話テーマ抽出
- メインテーマ・サブテーマを自動分類
- キーワードタグ生成（最大10タグ）

#### F-007: 感情分析
- 発言ごとの感情スコア算出（ポジティブ/ネガティブ/ニュートラル）
- 感情の時系列グラフデータ生成

#### F-008: 名言抽出
- 会話中の印象的な発言を自動抽出（最大5件）
- SNSシェア用テキスト自動生成

#### F-009: AI知識人アバターコメント生成
- 会話内容に対するAIコメンテーターのセリフ生成
- キャラクター設定: 博識なかわいいキャラクター複数体
- Gemini APIでキャラクター別のセリフトーン制御

#### F-010: 字幕生成
- SRT / WebVTT形式の字幕ファイル生成
- 話者名付き字幕
- 動画への字幕焼き込み対応

#### F-011: 動画生成
- FFmpeg + Remotionを使用した動画レンダリング
- アバターアニメーション + 字幕 + 背景
- 出力形式: MP4 (H.264, 1080p)
- 動画長: 元音声の長さに準拠（Shorts用は60秒）

#### F-012: YouTube投稿
- YouTube Data API v3を使用した自動投稿
- タイトル・説明文・タグの自動生成
- サムネイル自動生成

#### F-013: Shorts化
- 名言・ハイライトシーンから60秒以内のShorts動画を自動生成
- 縦型動画(9:16)対応

## 3. 非機能要求

### 3.1 パフォーマンス要件
| 指標 | 目標値 |
|---|---|
| 文字起こし速度 | 音声1分あたり30秒以内（Whisper medium） |
| 動画生成時間 | 10分音声 → 動画生成 10分以内 |
| APIレスポンス | 95パーセンタイルで3秒以内 |

### 3.2 可用性要件
- MVP段階: 個人利用のためSLA規定なし
- Docker Composeで単一ホスト運用
- Supabaseの無料枠を活用

### 3.3 セキュリティ要件
- APIキーは環境変数で管理（.envファイル）
- Google OAuth 2.0によるユーザー認証
- Supabase RLSによるデータアクセス制御
- アップロードファイルのウイルススキャン（MVP後）

### 3.4 コスト要件
| サービス | 利用枠 | 月額目標 |
|---|---|---|
| Gemini API | 無料枠 (1M tokens/月) | $0 |
| Supabase | 無料枠 (500MB DB, 1GB Storage) | $0 |
| YouTube Data API | 無料枠 (10,000 units/日) | $0 |
| VPS/サーバー | 個人PC or 低コストVPS | ~$5 |
| OpenAI Whisper | ローカル実行 | $0 |
| **合計** | | **~$5/月** |

### 3.5 スケーラビリティ要件
- Celeryによる非同期タスクキューで処理をスケール可能に設計
- Redisをブローカーとして使用
- 将来的にはKubernetes移行を想定したDockerコンテナ化

---

## 4. 制約条件

### 4.1 技術制約
- Google Meet の公式録音APIは存在しないため、画面録画ツール連携 or ブラウザ拡張経由で音声取得
- Remotionは Node.js ランタイムが必要（Pythonバックエンドからサブプロセス実行）
- Whisperのlargeモデルはメモリ8GB以上を要求

### 4.2 法的制約
- 会話参加者全員の録音同意が必要
- YouTubeコンテンツポリシー遵守
- 個人情報保護法に基づくデータ管理

### 4.3 ビジネス制約
- 個人開発フェーズは有料プラン不使用を原則
- MVPは1人のエンジニアで3ヶ月以内に完成
