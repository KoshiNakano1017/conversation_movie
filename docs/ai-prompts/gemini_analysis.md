# Gemini分析プロンプトの設計意図

**更新メモ（2026-05-23）**: 入力トランスクリプトは音声起こし結果だけでなく、ユーザーがアップロードしたトランスクリプトファイル由来の本文も対象とする。

## 目的
会議トランスクリプトから要約・テーマ・名言・YouTubeメタデータを1回のAPIコールで取得する。

## 設計上の選択と理由

### 1回コール方針
- 複数のプロンプトに分割すると API コスト・レイテンシが増大する
- Gemini 1.5 Flash の 1M トークンコンテキストで1回で十分

### JSON強制出力
- `response_mime_type="application/json"` を指定してパース安定化
- バックアップとして正規表現でJSON部分を抽出するフォールバックを用意

### 言語パラメータ
- 日本語・英語混在コンテンツに対応
- 分析結果の言語はユーザーの入力言語に合わせる

## 代替案と却下理由
- GPT-4o-mini → Gemini Flash の方が無料枠が大きく個人開発に向いている
- Function Calling → プロンプト内のJSON指定で十分、複雑性を避けた
